# Learning Handbook

## Core Goal

Become an AI master champion by building real systems step by step in this repository.

## Why This Project Exists

This project turns learning into execution.
You study concepts, implement them in the right module, run examples, and validate behavior with tests.

## Where To Learn And Practice

1. `python-learning/`: concept learning and revision material.
2. `backend/`: production-style implementation practice.
3. `examples/`: fast, practical end-to-end execution.
4. `tests/`: correctness and reliability checks.

## Step-By-Step Learning Loop

1. Learn: read concept material from `python-learning/`.
2. Build: implement or improve one focused capability in `backend/`.
3. Run: validate behavior through `examples/`.
4. Verify: add/update tests in `tests/unit/` or `tests/integration/`.
5. Record: capture insights and next actions in docs/memory.

## Current Mastery Tracks

1. Core reliability and engineering discipline.
2. Multi-provider LLM client engineering.
3. Retrieval and ranking system design.
4. Agentic orchestration and tool use.
5. Operational readiness: logging, errors, retries, and lifecycle.

## Milestone Definition

A milestone is complete when:
1. Feature works in code.
2. Example demonstrates it clearly.
3. Tests cover happy-path and failure-path behavior.
4. Docs reflect the final design and learning outcome.

## Project Vision Statement

This is your personal AI systems dojo.
Every iteration should move you closer to expert-level AI architecture, implementation, and operations.
